# 60.3 — The OSRS cache

XRSPS does not ship with the OSRS game cache. The cache — a read-only bundle of models, maps, items, NPCs, textures, animations, and CS2 scripts — is downloaded from [OpenRS2 Archive](https://archive.openrs2.org) at first run.

## What's in the cache

The cache is a set of files:

- **`main_file_cache.dat2`** — one giant file containing every asset.
- **`main_file_cache.idxN`** — index files (0-21), each pointing into `.dat2` for a given asset category (maps, models, items, NPCs, etc.).
- **`main_file_cache.idx255`** — the master index (index of indexes).
- **`keys.json` (XTEA)** — per-region XTEA decryption keys for map data. Maps are encrypted with a per-region key; the cache alone can't be read without these.

Both the client and the server read from the same extracted cache on disk.

## Cache version pinning

The current cache version is a single string in the repo's `target.txt` file:

```
osrs-237_2026-03-25
```

`scripts/ensure-cache.ts` reads this file and matches it against the entries in `caches/caches.json` to find the right OpenRS2 archive id. If your clone contains `caches/<cache-version>/`, no download is performed. Otherwise, the ensurer downloads and extracts it.

To bump the cache version:

1. Find the new OpenRS2 archive id at `https://archive.openrs2.org/caches`.
2. Update `target.txt` with the new name.
3. Update `caches/caches.json` with the new archive id (`{ "name": "osrs-238_...", "id": 12345 }`).
4. Delete `caches/<old-version>/`.
5. Run `bun run ensure-cache`.

After the cache updates, rebuild the server's collision cache:

```sh
bun run server:build-collision
```

This processes the new map data into `server/cache/collision/<region>.bin` files.

## File layout after extraction

```
caches/
├── caches.json               OpenRS2 archive metadata (hand-curated)
├── .cache-download.lock      lock file, auto-cleaned
└── osrs-237_2026-03-25/      extracted cache dir
    ├── main_file_cache.dat2
    ├── main_file_cache.idx0
    ├── ...
    ├── main_file_cache.idx21
    ├── main_file_cache.idx255
    └── keys.json
```

The cache directory is referenced by both the client (`src/cache/`) and the server (`server/src/cache/`). Each side opens the `.dat2` file and parses indexes independently.

## Download flow

`scripts/ensure-cache.ts`:

1. Reads `target.txt` to get the target version name.
2. Looks for `caches/<version>/main_file_cache.dat2`.
3. If missing, acquires a lock file (`caches/.cache-download.lock`) so two parallel invocations don't both download at the same time. Stale locks older than 10 minutes are removed.
4. Fetches the archive metadata from OpenRS2 (`https://archive.openrs2.org/caches`).
5. Finds the archive id in `caches/caches.json`.
6. Downloads the zipped cache to a temp location.
7. Extracts to `caches/<version>/`.
8. Releases the lock.

Pass `--server` to preload the server-only files (including a collision cache check). Without it, the script only fetches what the client needs.

## Client-side cache IO

The client reads the cache through `src/cache/CacheSystem.ts`. It is backed by IndexedDB in the browser (the `.dat2` file is downloaded from HTTP on first visit and cached in IDB for subsequent loads). Each `RenderDataWorker` (scene builder worker) opens its own read-only view.

See [10.1 — Cache subsystem](../10-client/01-cache.md) for the full client-side picture.

## Server-side cache IO

The server reads the cache through `server/src/cache/` (largely mirroring the client's parser). Key helpers:

- **`CacheEnv`** — holds the opened cache and provides typed loaders (`loadNpcType`, `loadObjType`, `loadLocType`, `loadMap`, `loadAnim`).
- **`initCacheEnv(cacheDir)`** — called from `server/src/index.ts` at boot.
- **`MapCollisionService`** — reads the baked collision cache from `server/cache/collision/` (generated by `build-collision-cache.ts`).

Collision data is pre-baked because walking it out of `.dat2` on demand is slow. The pre-bake runs once after cache updates.

## Size and bandwidth

A full extracted cache is roughly 1 GB. The zipped download is around 300-500 MB. This matters for:

- **CI** — you probably don't want to download the cache on every build. Cache the `caches/` directory between CI runs.
- **First client load** — the client downloads the cache from its static host and writes to IndexedDB. Expect a slow first load.

## Licensing

## Canonical facts

- **Version file**: `target.txt`.
- **Archive metadata**: `caches/caches.json`.
- **Cache root**: `caches/<version>/`.
- **Lock file**: `caches/.cache-download.lock` (10-minute stale timeout).
- **Ensurer**: `scripts/ensure-cache.ts`.
- **Downloader source**: `https://archive.openrs2.org`.
- **Server cache loader**: `server/src/cache/` + `CacheEnv`.
- **Client cache loader**: `src/cache/CacheSystem.ts`.
- **Collision bake**: `server/cache/collision/` (generated by `bun run server:build-collision`).
- **Rule**: client and server read from the same extracted cache directory.
